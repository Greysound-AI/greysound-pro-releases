## Greysound Pro 0.1.4

- Fixed the virtual MIDI keyboard so it always feeds armed or input-monitored instrument tracks without appearing as a separate device, while physical MIDI devices and All MIDI Ins remain selectable independently.
- Fixed the desktop bridge rejecting virtual-keyboard note messages, restored live synth monitoring and meters, and fixed empty instrument tracks dropping injected MIDI before it reached their instrument.
- Fixed virtual MIDI notes not appearing during recording and ensured completed MIDI takes retain their notes and regions.
- Migrated stale virtual-keyboard and All MIDI Ins session preferences so they no longer produce false missing-device warnings.
- Prevented held virtual-keyboard keys from triggering the macOS invalid-key alert while preserving normal operating-system shortcuts.
- Stopped sustained MIDI notes and instrument audio immediately when playback is paused or stopped.
- Fixed a built-in instrument teardown crash caused by parameter lifetime ordering.
- Fixed manual saves failing after a recording lost its one-shot frontend notification. Greysound Pro now recovers the recorded file's local session path so the save completes and the dirty marker clears.
- Fixed cloud sessions failing to initialize after cloning by preserving their cloud ID locally, reliably detecting downloaded `.gss` files, and cleaning up incomplete clones instead of leaving broken sessions behind.
- Fixed cloned and older sessions opening without waveforms. Missing or corrupt waveform sidecars are now regenerated from local audio, persisted, and loaded before the DAW becomes available.
- Improved cloud-clone and save diagnostics with the affected paths, directory contents, and JavaScript error stacks.
- Redesigned the crash-report dialog to match the DAW and removed implementation-specific wording.
- Improved desktop crash reports with structured native stack frames on macOS, Windows, and Linux, plus clearer WebKit renderer watchdog context when a renderer stack is unavailable.
- Fixed CoProducer being allowed to call Add MIDI Notes without any notes. The tool now requires non-empty raw notes, or non-empty patterns with at least one placement, while continuing to support both forms together.
